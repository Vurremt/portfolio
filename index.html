Game Engine - Gutter Engine

Gutter Engine is a 3D game engine developed in C++, designed to integrate realistic rendering, credible physics simulation, and a modular architecture based on the Entity-Component-System model. It offers a user interface via ImGui and allows programming independent games through a dedicated API.

The project was carried out as a final year project by 5 students, including myself. The goal was to design a functional and flexible engine capable of supporting a demo game — in our case, a bowling simulation — while exploring current standards in 3D development.

The engine includes components for PBR rendering, light and shadow management, realistic physics with PhysX, and behavior programming via C++ scripts. For more technical and visual details, the repository contains a complete README and illustrated examples.

UE5 Test - ThousandPasts

My current project on Unreal Engine 5 is a tech demo of a puzzle game with a time loop mechanic.

All of the player's actions are recorded while he plays in a loop, and in the next loop, a "ghost" from his past will repeat the same actions, such as moving, interacting with objects, etc. This will allow player to unlock passages by activating multiple levers simultaneously, free up sections of the map inaccessible on his own alone, and so on. All of this is within a single-player puzzle game.

Primarily developed using Blueprints, this project aims to train me in Unreal Engine 5 using C++, a language I already practice a lot, while simultaneously producing a game design and level design project that I enjoy.

Game Jam - "Sor(t)s !"

I've participated in numerous Game Jams, the most recent taking place in October 2025. The theme was "Sor(t)s", a pun between "Get Out!" and "Spells" in french, leaving room for interpretation.

We created a management game set in a fantasy kingdom, where the goal was to judge criminals. Sentences were in the form of magical spells, and everything hinged on banishing the criminal by shouting "Get out!", hence the pun (in french). Depending on the criminal's magical race, the other races had expectations that had to be met as best as possible, under penalty of being deposed and executed.

Using a pixel art style and a Python base (with the PyGame library), we developed a functional game in two days, which we are very proud of.
